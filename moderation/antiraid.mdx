## Configuring the antiraid system

Firstly, toggle on antiraid using the `,antiraid toggle on` command.

## Setting up antiraid modules

<Warning>
  Antiraid is **not** like [antinuke](/moderation/antinuke) and can be edited by **anyone** with the `administrator` permission.
</Warning>

Antiraid consists of several modules that monitor join events to prevent raids.

### Available Modules

<AccordionGroup>
  <Accordion title="Mass Join">
    Detects when a large number of members join your server in a short period.
    - **Usage:**
      <CodeGroup>
      ```javascript Command
      ,antiraid module massjoin threshold (number)
      ```
      ```javascript Example
      ,antiraid module massjoin threshold 10
      ```
      </CodeGroup>
  </Accordion>
  <Accordion title="Avatar">
    Detects members joining with the default Discord profile picture.
    - **Usage:**
      <CodeGroup>
      ```javascript Command
      ,antiraid module avatar toggle (on/off)
      ```
      ```javascript Example
      ,antiraid module avatar toggle on
      ```
      </CodeGroup>
  </Accordion>
  <Accordion title="New Accounts">
    Detects members joining with accounts created very recently.
    - **Threshold:** Age in days (e.g., 3 days).
    - **Usage:**
      <CodeGroup>
      ```javascript Command
      ,antiraid module newaccounts threshold (days)
      ```
      ```javascript Example
      ,antiraid module newaccounts threshold 3
      ```
      </CodeGroup>
  </Accordion>
  <Accordion title="Mass Mention">
    Detects and prevents mass mention raids from incoming members.
    - **Usage:**
      <CodeGroup>
      ```javascript Command
      ,antiraid massmention (on/off) [threshold] [action]
      ```
      ```javascript Example
      ,antiraid massmention on 5 mute
      ```
      </CodeGroup>
    <Tip>This module is part of greed premium.</Tip>
  </Accordion>
  <Accordion title="Unverified Bots">
    Prevents unverified bots from joining your server.
    - **Usage:**
      <CodeGroup>
      ```javascript Command
      ,antiraid unverifiedbots (on/off) [action]
      ```
      ```javascript Example
      ,antiraid unverifiedbots on kick
      ```
      </CodeGroup>
    <Tip>This module is part of greed premium.</Tip>
  </Accordion>
  <Accordion title="Username">
    Filters members joining with specific username patterns.
    - **Usage:**
      <CodeGroup>
      ```javascript Command
      ,antiraid username add (pattern)
      ```
      ```javascript Example
      ,antiraid username add raid-bot-.*
      ```
      </CodeGroup>
    <Tip>This module is part of greed premium.</Tip>
  </Accordion>
</AccordionGroup>

### Module Configuration

You can configure each module using the following commands:

<CodeGroup>
```javascript Command
,antiraid module (module) toggle (on/off)
```

```javascript Example
,antiraid module massjoin toggle on
```
</CodeGroup>

<CodeGroup>
```javascript Command
,antiraid module (module) threshold (value)
```

```javascript Example
,antiraid module massjoin threshold 10
```
</CodeGroup>

<CodeGroup>
```javascript Command
,antiraid module (module) punishment (action)
```

```javascript Example
,antiraid module avatar punishment kick
```
</CodeGroup>

## Username Patterns

<Tip>
  Username patterns are part of greed premium.
</Tip>

You can manage username patterns to block specific types of accounts from joining.

<CodeGroup>
```javascript Command
,antiraid username add (pattern)
```

```javascript Example
,antiraid username add raid-bot-.*
```
</CodeGroup>

<CodeGroup>
```javascript Command
,antiraid username remove (pattern)
```

```javascript Example
,antiraid username remove raid-bot-.*
```
</CodeGroup>

<CodeGroup>
```javascript Command
,antiraid username list
```

```javascript Example
,antiraid username list
```
</CodeGroup>

<CodeGroup>
```javascript Command
,antiraid username action (kick/ban)
```

```javascript Example
,antiraid username action ban
```
</CodeGroup>

## Lockdown

If your server is under an active raid, you can use the lockdown command to quickly restrict access.

<CodeGroup>
```javascript Command
,antiraid lockdown (on/off)
```

```javascript Example
,antiraid lockdown on
```
</CodeGroup>

## Advanced Lockdown

```javascript Example

```
</CodeGroup>

<CodeGroup>


## Logging

It is highly recommended to set a log channel for antiraid to monitor its actions.

<CodeGroup>
```javascript Command
,antiraid logs (channel)
```

```javascript Example
,antiraid logs #raid-logs
```
</CodeGroup>

## Whitelisting

You can whitelist specific users or roles from being affected by the antiraid.

<CodeGroup>
```javascript Command
,antiraid whitelist add (user/role)
```

```javascript Example
,antiraid whitelist add @Moderator
```
</CodeGroup>

<CodeGroup>
```javascript Command
,antiraid whitelist remove (user/role)
```

```javascript Example
,antiraid whitelist remove @fe3murs
```
</CodeGroup>
